# C4 — Arquitectura de Luces Errantes

El modelo **C4** describe la arquitectura en cuatro niveles de zoom progresivo.

| Nivel | Pregunta que responde |
|---|---|
| **L1 — Contexto** | ¿Quién usa el sistema y con qué servicios externos interactúa? |
| **L2 — Contenedores** | ¿Qué procesos o artefactos desplegables componen el sistema? |
| **L3 — Componentes** | ¿Qué piezas internas tiene cada contenedor? |
| **L4 — Código** | ¿Cómo se implementa cada componente? → ver código fuente |

---

## L1 — Contexto del Sistema

```
  ┌────────────────────┐              ┌──────────────────────┐              ┌──────────────────────────────┐
  │     👤 Jugador      │              │    ☁️  Cloudflare     │              │      🎮 Luces Errantes        │
  │────────────────────│              │──────────────────────│              │──────────────────────────────│
  │  Navegador web     │              │  CDN                 │              │  Juego multijugador          │
  │  PC o móvil        │──── HTTPS ──►│  Terminación HTTPS   │──── HTTP ───►│  en tiempo real              │
  │                    │              │  Protección DDoS     │              │  Svelte 5 + Go + WebSocket   │
  │                    │              │  WebSockets: ON ✓    │              │                              │
  └────────────────────┘              └──────────────────────┘              └──────────────────────────────┘
```

| Elemento | Tipo | Descripción |
|---|---|---|
| Jugador | Actor | Accede desde cualquier navegador moderno |
| Cloudflare | Sistema externo | Gestiona HTTPS, CDN y DDoS. Requiere **WebSockets activado** en Dashboard → Network |
| Luces Errantes | Sistema principal | Frontend + backend como contenedores Docker en un VPS |

                    │  │                                          │   red interna)        │ │    │

```

| Contenedor | Imagen base | Puerto | Responsabilidad |
|---|---|---|---|
| Nginx Proxy Manager | jc21/nginx-proxy-manager | 80 / 443 (host) | TLS, routing por dominio, WebSocket forwarding |
| Frontend | nginx:alpine | 8060 → 80 | Sirve build de Svelte + proxy inverso de `/ws` al backend |
| Backend | alpine + binario Go | 8080 (solo interno) | Motor del juego, WebSocket, broadcast de estado |

> El puerto 8080 del backend **nunca se expone al host**. Solo es accesible desde el frontend a través de la red interna Docker, usando el hostname `backend`.

```

| Componente | Archivo | Función |
|---|---|---|
| Static File Server | `nginx.conf location /` | Sirve `dist/` con fallback a `index.html` para rutas SPA |
| WebSocket Proxy | `nginx.conf location /ws` | Proxy al backend con headers `Upgrade` + `Connection: upgrade` |
| Pantalla Login | `App.svelte` | Canvas animado de fondo + formulario → inicia conexión WS |
| Cliente WebSocket | `App.svelte joinGame()` | Abre `ws(s)://host/ws`, envía join, despacha mensajes |
| Interpolador | `App.svelte updateInterpolation()` | Interpola posiciones entre el tick anterior y el nuevo (50ms → 60fps) |
| Caché de obstáculos | `App.svelte buildObstacleCache()` | Dibuja obstáculos en canvas offscreen; rebuild solo cuando la cámara se mueve >100px |
| UI Overlay | `App.svelte drawGame()` | Ranking, info del jugador, minimapa y controles sin transformación de cámara |

## L3 — Componentes del Backend

```
  Contenedor: Backend (alpine + binario Go)
  ┌─────────────────────────────────────────────────────────────────────────────────┐
  │                                                                                 │
  │   main.go — Punto de entrada                                                    │
  │   ┌────────────────────────────────────────────────────────────────────────┐   │
  │   │  HTTP Server :8080                                                      │   │
  │   │  HandleFunc /ws  ──►  WebSocket Upgrader (gorilla/websocket)           │   │
  │   │                        CheckOrigin: acepta cualquier origen             │   │
  │   └───────────────────────────────────────────┬────────────────────────────┘   │
  │                                               │ crea Client{} por conexión     │
  │                                               ▼                                │
  │   hub.go — Por cada cliente conectado                                          │
  │   ┌────────────────────────────────────────────────────────────────────────┐   │
  │   │                                                                        │   │
  │   │   Client{ conn, send chan[]byte, player }                              │   │
  │   │                                                                        │   │
  │   │   ┌──────────────────────────────┐   ┌───────────────────────────┐   │   │
  │   │   │  📥 readPump()               │   │  📤 writePump()            │   │   │
  │   │   │  [goroutine por cliente]     │   │  [goroutine por cliente]   │   │   │
  │   │   │  ─────────────────────────  │   │  ─────────────────────── │   │   │
  │   │   │  Lee frames WS del browser  │   │  Escribe desde chan send   │   │   │
  │   │   │  Decodifica JSON            │   │  Ping cada 54s (keepalive) │   │   │
  │   │   │  Rutea por tipo de mensaje: │   │  WriteDeadline: 10s        │   │   │
  │   │   │  join   → game.join chan    │   └───────────────────────────┘   │   │
  │   │   │  input  → player.SetDir()  │                                    │   │
  │   │   │  target → player.SetTarget()                                    │   │
  │   │   │  chat   → game.chat chan    │                                    │   │
  │   │   │  ReadDeadline: 60s (reset  │                                    │   │
  │   │   │  en cada Pong recibido)    │                                    │   │
  │   │   └──────────────────────────┘                                     │   │
  │   └────────────────────────────────────────────────────────────────────┘   │
  │                          │ canales: register · unregister · join · chat      │
  │                          ▼                                                   │
  │   game.go — Goroutine única global                                           │
  │   ┌────────────────────────────────────────────────────────────────────┐   │
  │   │                                                                    │   │
  │   │   Run() — select sobre todos los canales                          │   │
  │   │   ┌──────────────────────────────────────────────────────────┐   │   │
  │   │   │  Eventos de canal            │  Timers                   │   │   │
  │   │   │  ─────────────────────────  │  ─────────────────────    │   │   │
  │   │   │  register   → añadir cliente│  ticker 50ms              │   │   │
  │   │   │  unregister → quitar cliente│    └─► tick()             │   │   │
  │   │   │  join   → crear jugador     │                           │   │   │
  │   │   │          enviar welcome     │  spawnTimer 2–5s aleatorio│   │   │
  │   │   │  chat   → broadcast mensaje│    └─► spawnLight()        │   │   │
  │   │   └──────────────────────────────────────────────────────────┘   │   │
  │   │                                                                    │   │
  │   │   tick() — cada 50ms                                              │   │
  │   │   ┌──────────────────────────────────────────────────────────┐   │   │
  │   │   │  1. Update posición de todos los jugadores (dt = 1/20s)  │   │   │
  │   │   │  2. DeflectFromObstacle → empuja jugadores fuera          │   │   │
  │   │   │  3. Colisión jugador ↔ luz (distancia euclidiana)        │   │   │
  │   │   │     → jugador crece: luz.size × 0.3                      │   │   │
  │   │   │  4. Serializar estado JSON (players + lights)            │   │   │
  │   │   │  5. Calcular ranking top 10 por tamaño                   │   │   │
  │   │   │  6. Broadcast a todos los clientes via chan send          │   │   │
  │   │   └──────────────────────────────────────────────────────────┘   │   │
  │   │                                                                    │   │
  │   │   Al iniciar: generateObstacles()                                 │   │
  │   │   15–25 obstáculos · radio 40–120px · sin solapamiento            │   │
  │   │   Se envían al cliente en el mensaje welcome                      │   │
  │   └────────────────────────────────────────────────────────────────┘   │
  │                          │                                               │
  │                          ▼                                               │
  │   player.go — Entidad Jugador                                           │
  │   ┌────────────────────────────────────────────────────────────────┐   │
  │   │   Player{ sync.RWMutex, ID, Name, X, Y, Size, Color }         │   │
  │   │                                                                 │   │
  │   │   Update(dt)          velocidad = 200 / (1 + (size-15)×0.008) │   │
  │   │   SetDirection(dx,dy) modo WASD / teclas de flecha             │   │
  │   │   SetTarget(x,y)      modo click-to-move                       │   │
  │   │   DeflectFromObstacle empuja al jugador fuera del obstáculo    │   │
  │   │   Grow(amount)        aumenta tamaño al recolectar luz         │   │
  │   │                                                                 │   │
  │   │   Nota: usa sync.RWMutex porque readPump() (otra goroutine)   │   │
  │   │   llama a Set*() mientras el game loop lee el estado          │   │
  │   └────────────────────────────────────────────────────────────────┘   │
  └─────────────────────────────────────────────────────────────────────────┘
```

| Componente | Archivo | Goroutines | Responsabilidad |
|---|---|---|---|
| HTTP Server | `main.go` | 1 (main) | Escucha en `:8080`, único endpoint `/ws` |
| WebSocket Upgrader | `main.go` | — | Convierte HTTP → WS, crea `Client{}` |
| readPump | `hub.go` | 1 por cliente | Lee frames, decodifica JSON, envía a canales |
| writePump | `hub.go` | 1 por cliente | Escribe desde `send`, ping cada 54s |
| Game Loop | `game.go` | 1 global | Único modificador del estado — sin data races |
| tick | `game.go` | — | move → deflect → collide → rank → broadcast |
| generateObstacles | `game.go` | — | 15–25 obstáculos al iniciar, sin solapamiento |
| spawnLight | `game.go` | — | Crea luces fuera de obstáculos; inicial 15, máx. 40 |
| Player | `player.go` | — | Entidad con `sync.RWMutex` para acceso concurrente |

## Protocolo de Mensajes

### Cliente → Servidor

| Tipo | Campos | Cuándo |
|---|---|---|
| `join` | `name` (máx 20 chars) | Una vez al conectar |
| `input` | `dx`, `dy` (−1, 0 ó 1) | Cada keydown / keyup |
| `target` | `x`, `y` (coord. mundo) | Cada click en el canvas |
| `chat` | `text` (máx 200 chars) | Al enviar mensaje |

### Servidor → Cliente

| Tipo | Campos | Frecuencia |
|---|---|---|
| `welcome` | `id`, `mapW`, `mapH`, `obstacles[]` | 1 vez tras `join` |
| `state` | `players[]`, `lights[]`, `top[]` | 20 veces/seg |
| `chat` | `sender`, `text`, `timestamp` | Al recibir mensaje de cualquier jugador |

```

      │
      ├── 1. Verifica Docker instalado y daemon corriendo

      │           │
      │           ├── zen-garden/Dockerfile

| Archivo | Ubicación | Función |
|---|---|---|
| `deploy.sh` | raíz | Orquesta todo: checks → down → build → up → validate |

## Resumen — Cadena completa

```
  Jugador (browser)
       │
       │  wss://dominio.com/ws  (puerto 443)
       ▼
  Cloudflare
       │  WebSockets: ON  (Dashboard → Network → WebSockets)
       │  Proxy inverso → VPS
       ▼
  Nginx Proxy Manager  [contenedor Docker, VPS]
       │  Websockets Support: ON  (configuración del Proxy Host)
       │  Enruta dominio → localhost:8060
       ▼
  Frontend  [contenedor Docker, nginx:alpine]
       │  Puerto 8060 (host) → 80 (contenedor)
       ├── GET /      →  sirve build de Svelte (index.html + assets)
       └── GET /ws    →  proxy_pass http://backend:8080/ws
